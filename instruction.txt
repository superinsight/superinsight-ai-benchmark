## Process (Mandatory — Internal Only)
Before generating the final output, analyze the document internally:
1. **Extraction:** Identify all potential encounters.
2. **Inclusion Filtering:** A visit is valid if it has **at least** a **Facility Name** OR a **Provider Name** explicitly present in the text. If one is missing, still include the visit but write `Not specified` for the missing field. Do NOT invent or guess names — only use names explicitly stated in the source.
3. **Scope Filtering:**
   - Discard all legal events (depositions, legal reviews).
   - **CRITICAL EXCLUSION:** Discard "Provider Lists", "Directories", "Staff Updates", "Medical Provider List Updates", or "Summaries of Treating Providers". These are administrative lists, NOT clinical visits.
   - Discard non-medical events (billing, admin).
4. **Sorting:** Sort valid visits chronologically.
5. **Deduplication:** Ensure distinct appointments on the same day are listed as separate entries. **CRITICAL:** If a patient sees **different providers** at the **same facility** on the **same day** (e.g., Dr. Smith and Dr. Jones both at City Hospital), these must be treated as **TWO SEPARATE** visits with their own `###` headers. Do not merge them.

## Input Criteria (Filtering Rules)
1. **Inclusion:** A visit is valid if **at least one** of `Facility Name` or `Provider Name` is explicitly present in the text. If one is missing, write `Not specified` — do NOT fabricate names.
2. **Medical Only:** Include only clinical/medical visits where a patient was seen or treated.
   - **Exclude:** Legal events, pharmacy fills, administrative calls, scheduling notes, billing/insurance notes.
   - **Exclude:** Lists of providers, phone directories, or "Updates to Medical Provider List".
3. **Date Accuracy:** Use "Date of Service" only.
   - If specific day is missing, use the format MM/YYYY (e.g., "02/2024").
   - **NEVER** output the literal string "MM/YYYY". If no date is found, exclude the entry.

## Output Rules (Hard Constraints)
- Output **ONLY** the final “Medical Chronology” section.
- **Formatting:** Ensure **STRICT** markdown structure. Every bullet point must start on a NEW LINE. Do not output run-on paragraphs.
